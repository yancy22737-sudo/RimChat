# RimWorld 任务类型限定条件分析

本文档总结了原版及 DLC 任务脚本（QuestScriptDefs）中常见的派系、NPC 及环境限定条件，用于指导 AI 任务生成的合理性校验。

## 1. 派系身份限定 (Faction Constraints)

| 限定条件 | 影响范围 | 说明 |
| :--- | :--- | :--- |
| **禁止永久敌对** | 绝大多数外交任务 | 机械族、虫群、永久敌对海盗无法发起租借、和平会谈或贸易请求。 |
| **科技等级要求** | PawnLend (租借) | 发起派系必须达到 `Industrial` (工业时代) 或以上。部落派系不适用。 |
| **帝国专属** | 授爵仪式、皇家升天 | 严格锁定至 `Empire` 派系，且要求玩家具备特定的爵位等级。 |
| **好感度阈值** | 招待任务 (Hospitality) | 点数过低时会排除帝国，防止早期玩家无法满足其高标准需求。 |

## 2. 发起人约束 (Asker Constraints)

通过 `QuestNode_GetPawn` 节点进行筛选：

- **必须为领袖 (mustBeFactionLeader)**：
  - 代表任务：`BanditCamp` (土匪营地)、`PeaceTalks` (和平会谈)。
  - 逻辑：任务代表派系正式外交行为，必须由派系首领签发。
- **必须具备头衔 (mustHaveRoyalTitle)**：
  - 代表任务：`PawnLend` (租借)、部分 `ThreatReward`。
  - 逻辑：允许派系内部的高级贵族作为发起人。
- **敌对权重修正 (HostileWeight: 0.15)**：
  - 大多数随机任务中，敌对派系作为发起人的概率被大幅降低（仅为盟友的 15%）。

## 3. 任务触发阈值 (Trigger Thresholds)

- **最小威胁点数 (rootMinPoints)**：
  - `BanditCamp` 要求 350 点。
  - 财富值过低的早期定居点不会收到高级战斗任务。
- **进度分要求 (rootMinProgressScore)**：
  - `PeaceTalks` 要求 10 分。
  - 确保玩家在建立稳定基地前不会被复杂外交打扰。

## 4. 环境与地理约束 (Environmental Constraints)

- **距离衰减 (Distant Fight)**：
  - 目标点距离过远会大幅下调奖励，甚至导致任务无法生成。
- **主基地要求 (Home Map)**：
  - 绝大多数任务要求玩家必须拥有一个已加载的“主地图”。在全员远征或全员迁徙状态下，系统会暂停剧情任务推送。

## 5. 典型任务约束对照表

| 任务 DefName | 核心派系限制 | 发起人要求 | 目标/威胁要求 |
| :--- | :--- | :--- | :--- |
| `BanditCamp` | 非永久敌对 | 派系领袖 | 目标派系必须与发起方敌对 |
| `ItemStash` | 非敌对派系 | 60% 领袖 / 40% 无 | 85% 几率伴随动态威胁 |
| `PeaceTalks` | 允许敌对(非永久) | 派系领袖 | 领袖必须处于“安全”状态 |
| `PawnLend` | 工业时代以上 | 领袖或贵族 | 要求玩家殖民地有富余劳动力 |
| `Hospitality` | 多样化 | 难民领袖/官员 | 对卧室、食物、心情有硬性要求 |

## 开发者建议
在通过 AI 生成任务时，务必调用 `ValidateAndFixQuestDef` 方法进行预校验。如果 AI 请求的任务不满足上述硬性条件（例如部落派系发起租借任务），系统应自动回退到通用的 `ThreatReward_Raid_MiscReward`。
