Walkthrough: Empire Quest Bug Fix
Problem
Normal AI factions (e.g. tribal faction "贝科尔氏族协约") generating ThreatReward_Raid_MiscReward resulted in quests attributed to the Empire — with Empire descriptions, Royal Favor rewards, and Empire-titled senders.

Root cause: The quest's XML uses QuestNode_GetPawn with mustHaveRoyalTitleInCurrentFaction=true, which internally selects an Empire pawn as asker. We initially tried to protect the Slate variables via Harmony, but RimWorld uses a generic Slate.Set<T> method which Harmony cannot patch without throwing a Specified method is not supported error.

The Solution: Two-Layer Defense
Since we cannot patch the generic Slate.Set<T>, we implemented a robust two-layer defense that completely blocks non-Empire factions from attempting to generate Empire-related quests in the first place.
