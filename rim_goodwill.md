## 三、自然好感度回归机制

### 3.1 温和派系（文明外来者联盟/温和部落）
- **自然范围**：-50 ~ 50
- **超过50时**：每天下降0.4
- **低于-50时**：每天上升0.2

### 3.2 凶猛派系（粗暴外来者联盟/凶猛部落）
- **自然范围**：-100 ~ -80
- **超过-80时**：每天下降0.4

## 四、与RimDiplomacy模组配置对比

### 4.1 基础消耗对比

| 行为 | 原版 | RimDiplomacy当前 | 差异说明 |
|------|------|-----------------|---------|
| 请求商队 | -15 | -8 | 我们的基础值更低，但有5维关系修正 |
| 请求军事援助 | -25 | -15 | 同上 |
| 请求医疗援助 | - | -10 | 原版无此独立选项 |
| 请求资源援助 | - | -12 | 原版无此独立选项 |
| 赠送礼物 | +1/40-160银 | +8 | 我们的固定值更高 |
| 分享情报 | - | +5 | 原版无此独立选项 |
| 履行承诺 | - | +10 | 原版无此独立选项 |

### 4.2 系统差异说明

1. **原版系统**：
   - 固定消耗值
   - 简单冷却时间
   - 无关系值深度影响

2. **RimDiplomacy系统**：
   - 基础值 + 5维关系值动态修正
   - 每日上限（消耗50/收益30）
   - 行为冷却 + 每日限制双重机制
   - 关系阈值决定行为可用性

### 4.3 设计合理性说明

我们的消耗值普遍比原版低是合理的设计，因为：

1. **动态修正机制**：
   - 高信任值（60+）可减少30%消耗
   - 高亲密度（60+）可减少18%消耗
   - 正互惠值可进一步减少消耗
   - 综合减免最高可达70%

2. **限制机制更严格**：
   - 原版：仅冷却时间限制
   - 我们：冷却时间 + 每日上限 + 关系阈值

3. **5维关系系统**：
   - 原版：单一goodwill值
   - 我们：信任、亲密、互惠、尊重、影响五维评估

## 五、建议调整方案

### 5.1 可选调整（如需更接近原版）

```csharp
// 当前配置
public const int BaseCost_RequestCaravan = -8;      // 改为 -30
public const int BaseCost_RequestMilitaryAid = -15; // 改为 -50

public const int BaseCost_RequestResourceAid = -12; // 改为 -24

// 收益型行为
public const int BaseGain_SendGift = 8;             // 建议改为与银价值挂钩
```

### 5.2 保持当前设计的理由

1. **LLM对话系统更复杂**：需要更多交互空间
2. **5维关系值提供深度**：不是单一数值游戏
3. **双重限制机制**：防止过度刷好感
4. **动态体验**：不同派系关系体验差异更大
