版本更新日志
0.3.34
- 变更：RPG API 动作说明改为统一使用共享定义块动态注入，移除旧的 RPGApiGuidelines 固定注入路径。
- 优化：RPG 设置页移除旧“API说明”编辑分区，预览改为显示与运行时一致的动态动作定义文本。
0.3.33
- 变更：RPG 系统提示词在启用 API 时统一注入 AppendRPGApiDefinitions 动作定义段，不再拼接旧 RPGApiGuidelines 文本。
0.3.32
- 优化：RPG 对话 Job 在寻路过程中增加高频目标位移检测（每 15 ticks 检查一次），目标移动时即时重算路径，不再主要依赖“到达旧交互点后再跳转”。
- 优化：统一对话可开窗距离阈值常量，减少追踪与开窗判定不一致导致的停顿感。
0.3.31
- 修复：补齐 LeaderMemory JSON 持久化字段，新增保存/回读 lastDecayCheckTick、playerRelationValues、FactionMemoryEntry.firstContactTick/lastMentionedTick/relationHistory。
- 修复：移除领袖记忆写盘时的二次 50 条裁剪，避免与内存 200 条上限叠加导致历史信息过度丢失。
- 新增：RPG 对话按 NPC 独立文件持久化，路径为 save_data/<存档名>/rpg_npc_dialogues/npc_<pawnId>.json。
- 新增：NPC 档案记录回填到 RPG 运行态（人格 Prompt、五维 RPG 关系值、对话冷却截止 tick），提升跨读档一致性。
0.3.30
- 新增：新存档开局即为全部非玩家派系建立记忆基线快照（goodwill + 当前关系类型），避免首次交互时记忆空白。
- 新增：读档后支持从 `FactionDialogueSession.messages` 回填旧会话到派系记忆，实现中途接入存档的数据接管。
- 新增：回填写入幂等标记事件（`init-snapshot` / `session-backfill`），重复读档不会重复灌入同批数据。
- 优化：领袖记忆加载后自动补齐缺失的 5 维关系初值（按当前 goodwill 同步），提升旧档兼容稳定性。
0.3.29
- 新增：RPG-外交双向记忆链路（离图摘要 + 外交会话摘要），统一写入派系级长期记忆。
- 新增：Harmony Patch 接入 Pawn.ExitMap(bool, Rot4)，仅对“非玩家派系/人形/玩家家园地图/近期有RPG对话痕迹”的 Pawn 生成离图摘要。
- 新增：外交窗口关闭时按“是否有新增消息”生成会话摘要；无新增消息不生成。
- 新增：RPG 提示词动态注入派系共享记忆块（不覆盖 Persona），注入预算固定为“池上限20、注入6条、总长2200字符”。
- 新增：摘要策略采用“规则优先 + 低置信度触发 LLM 回退”；AI 不可用时自动保留规则摘要。
- 修复：领袖记忆 JSON 读写字段映射不一致问题，新增跨通道摘要字段并保持旧字段兼容解析。
0.3.28
- 新增：API 设置页底部显示“最近一次对话Token使用量：xxxx（低/中/高）”。
- 新增：记录外交与 RPG 会话的最近一次 Token 快照（prompt/completion/total、通道、时间戳）。
- 新增：优先解析响应 usage 字段，缺失时按“请求+响应文本（4字符≈1token）”回退估算并标注“估算”。
- 新增：中英文语言键 RimDiplomacy_LastDialogueTokenUsage* 与 RimDiplomacy_TokenLevel*。
0.3.27
- 修复：RPG 对话 Job 新增目标位置刷新回跳，目标 Pawn 持续移动时会重复追踪至近距离后再触发开窗。
- 修复：开窗前增加同图/已生成/距离二次校验，避免追踪过程中的错位触发。
- 优化：Job 增加可达性失败条件，减少目标移动导致的无效执行。
0.3.26

- 修复：TryGainMemory 失败提示新增“可用 defName 示例”反馈，便于模型与玩家快速纠正参数。
- 优化：RPG API 动作指引扩展 TryGainMemory 可用参数集合，并明确禁止使用 Chatted（应使用 Chitchat）。
- 修正：RPG 默认 JSON 模板中的 TryGainMemory 示例从 JoyFilled 更正为有效的 Chitchat。
